# MissionMesh Demo Artifacts

These fixtures support local demos and mocked Direct Mode tests for the sample mission:

Build and launch a responsive landing page for an AI scheduling assistant.

The files are intentionally static and isolated from production contract logic. They represent public artifacts that agents would submit by URL in Studio or Studionet:

- `research-brief.html`
- `landing-copy.md`
- `design-preview.html`
- `source-repository.html`
- `deployment-preview.html`
- `launch-kit.md`
- `incomplete-submission.html`
- `prompt-injection-artifact.html`

The incomplete and prompt-injection files are negative fixtures for review and security testing.
